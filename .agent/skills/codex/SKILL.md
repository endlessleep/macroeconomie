# Codex Logic & Premium Design Standards

This skill defines the "Matrix" for building premium, state-of-the-art revision fiches for Macroeconomics and Finance.

## 1. UI Aesthetics (The "Wow" Factor)
- **Glassmorphism:** Use `backdrop-filter: blur(12px)` and semi-transparent backgrounds for cards.
- **Color Palette:** Use HSL values for harmony. Avoid plain colors.
  - Primary (Macro): `hsl(217, 89%, 61%)` (Royal Blue)
  - Secondary (Finance): `hsl(333, 77%, 50%)` (Ruby)
  - Dark Mode: Surface `hsl(222, 47%, 11%)`, Text `hsl(213, 27%, 84%)`
- **Typography:** Always use Google Fonts (Inter, Outfit, or Roboto).
- **Animations:** Implement micro-interactions (hover scale, fade-in on scroll).

## 2. Logic & Coding (Clean Code)
- **Semantic HTML:** Use `<article>`, `<section>`, `<aside>`, `<details>`, `<summary>` instead of generic `<div>`.
- **CSS Architecture:** Use CSS Variables for all tokens. Keep `macro_style.css` as the single source of truth.
- **MathJax:** Use single `$` for inline and `$$` for display blocks. Always ensure rigorous notation.
- **Formatting:** Code must be indented with 4 spaces. Comments must be in French (as per USER preference) and line-by-line for complex logic.

## 3. Domain Logic (Macro & Finance)

  - 1: IS (Demand)
  - 2: PC (Supply/Phillips)
  - 3: MP (Taylor Rule)
  - 4: FE (Fisher Equation)
  - 5: Expectations (Adaptive/Anchored)
- **Causal Chains:** Always build step-by-step logic (1. Shock -> 2. Variable Reaction -> 3. Secondary Effect).
